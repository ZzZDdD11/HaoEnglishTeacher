# 影子跟读英语学习系统 — 设计文档

> 日期: 2026-07-17
> 状态: Draft

---

## 1. 产品概述

一个基于影子跟读（Shadowing）方法的英语发音学习系统。用户粘贴 YouTube/B站等视频链接，系统自动提取音频并生成字幕，用户逐句跟读模仿，系统自动对比发音差异并给出评分和纠音建议。

**与现有产品的区别**：市面上的语言助手（Duolingo、Speak、ELSA、ChatGPT Voice）都是对话交互模式，没有"视频影子跟读 + 自动发音对比纠正"的完整闭环。

---

## 2. 目标用户 & 平台

- **MVP 目标用户**：自学者（个人使用）
- **MVP 平台**：Web 应用（Next.js），后续扩展 iOS
- **v2 用户群**：增加英语老师/教练模式（布置作业、查看学生报告）

### BFF 层的职责

- **安全**：Python 服务不直接暴露公网，Next.js 作为网关转发请求
- **认证**：用户认证和 Session 管理由 Next.js 负责，Python 服务无状态
- **数据聚合**：为前端提供定制化的聚合接口，减少前端请求次数

## 4. 核心音频处理流水线

```
视频 URL → yt-dlp 下载 → ffmpeg 提取音频 (WAV 16kHz mono)
  → Whisper 转写 (带时间戳字幕)
  → 用户录音上传 → Azure Speech API 逐词发音打分
  → 返回评分 + 纠音建议
```

### MVP 简化版（v1）

> **不做**音素级强制对齐（MFA/gentle），用 Azure Speech Pronunciation Assessment API 直接逐词打分。

### v2 升级版

```
用户录音 → whisper 转写 → MFA 音素对齐 → 音素 embedding 余弦对比
  → 4 维度评估（音素相似度 / 时长节奏 / 音高曲线 / 音量包络）
  → LLM 生成通俗纠音建议
```

**逐句模式交互**：
1. 视频自动播放到当前句 → 暂停
2. 用户点击录音按钮 → 跟读 → 自动停止或手动停止
3. 显示波形对比和分数
4. 点击"下一句"继续

### 视图 3：反馈报告页

- 总分 + 三维度分数（发音/节奏/语调）

- 重点练习建议（中文纠音提示）
- 操作：重新练习弱项、查看历史进步曲线

## 6. 数据模型

### Material（练习素材）
```
id, user_id, source_type(youtube|bilibili|upload|builtin),
source_url, title, duration_seconds, language, difficulty_level,
transcript_json, status(processing|ready|error), created_at
```

### PracticeSession（练习记录）
```
id, user_id, material_id, mode(sentence_by_sentence|shadowing),
status(in_progress|completed), overall_score, pronunciation_score,
rhythm_score, intonation_score, completed_at
```

### SentenceAttempt（每句跟读）
```
id, session_id, sentence_index, audio_recording_url,
reference_audio_url, score, word_scores_json,
waveform_data_json, suggestions_json, created_at
```

### 存储选型

| 数据 | 存储 | 理由 |
|------|------|------|
| 结构化记录 | PostgreSQL | 关系查询灵活 |
| 评分/转写 JSON | PostgreSQL JSONB | 结构嵌套但需查询 |
| 音频文件 | 本地磁盘(MVP) → S3(v2) | 大文件，CDN 加速 |
| 任务队列 | Redis + Celery | 异步处理 |

## 7. API 设计

### Next.js BFF 层（面向前端）

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/materials` | POST | 提交视频 URL，触发后台处理 |
| `/api/materials` | GET | 素材列表（含处理状态） |
| `/api/materials/[id]` | GET | 素材详情 + 转写文本 |
| `/api/sessions` | POST | 开始一次练习 |
| `/api/sessions/[id]` | GET | 练习进度 |
| `/api/sessions/[id]/attempts` | POST | 上传跟读录音，返回评分 |
| `/api/sessions/[id]` | PATCH | 结束练习 |
| `/api/sessions/[id]/report` | GET | 完整报告 |
| `/api/users/me/stats` | GET | 学习统计/进步曲线 |

WebSocket: `/ws/session/[id]` – 实时推送视频处理进度

### Python FastAPI 层（内部服务）

| 端点 | 说明 |
|------|------|
| `POST /process/video` | 下载视频 + 提取音频 + Whisper 转写 |
| `GET /process/status/{task_id}` | 查询处理进度 |
| `POST /evaluate/pronunciation` | 用户音频 vs 参考音频 → 评分 |
| `GET /health` | 健康检查 |

## 9. MVP 范围

### MVP 做这些

- 粘贴 YouTube/B站链接 → 自动提取音频和字幕
- 逐句跟读模式（视频播放一句 → 暂停 → 用户跟读 → 逐词评分）
- Azure Speech API 逐词发音打分
- 基础纠音建议（基于 Azure API 返回的问题 + 规则模板生成中文提示）
- 波形对比可视化（Web Audio API 渲染）
- 练习报告页（逐句回顾 + 总分 + 建议）
- 游客模式（localStorage 存记录，无需注册）
- 本地 Docker Compose 一键启动

### MVP 不做（v2+）

- 内置素材库（内容运营成本高）
- 同步影子跟读模式
- 音素级对齐和评分（MFA / wav2vec）
- 用户注册登录系统
- iOS App
- 社交分享功能
- 上传视频文件（只做 URL 粘贴）

## 11. 风险 & 缓解

| 风险 | 缓解 |
|------|------|
| yt-dlp 受 YouTube 反爬限制 | 备选方案：用户自行下载视频上传 |
| Azure Speech API 成本随用量增长 | MVP 先控制用量；v2 逐步迁移到开源模型 |
| Whisper 转写精度不足（口音重/噪音） | 提供手动修正字幕的入口 |
| 浏览器录音质量受限 | 前端检测采样率，提示用户用耳机麦克风 |
| Web Audio API 波形渲染性能 | 只渲染当前可见区域，虚拟滚动 |
